Jinja guide

What is Jinja

Jinja is a fast, expressive, extensible templating engine. Special placeholders in the template allow writing code similar to Python syntax.

The template is passed data to render the final document. This allows sections of the code to be shown or hidden depending on user actions.>

Jinja uses various statements contained by {% raw %}{% ...%}.{% endraw %}

Official Jinja documentation.

If statements

An IF Statement is the cornerstone of any programming language. In Jinja it allows page content to be shown or hidden dependent on a variable set by a user action.

Simple IF Statement

If statements are started with an {% raw %}{% if ...%}.{% endraw %} and ended with {% raw %}{% endif %}.{% endraw %}

{% raw %}

{% if data['variable-1'] %}

... HTML to show if variable-1 is true ...

{% endif %}

{% endraw %}

Simple IF...ELSE Statement

You can also add an ELSE statement to an IF statement. This will allow you to show a section if a variable is one thing and a different section if it is something else.

{% raw %}

{% if data['variable-2'] == 'Option 1' %}

HTML if variable-2 is equal to 'Option-1' ...

{% else %}

HTML if variable-2 is not equal to 'Option-1' ...

{% endif %}

{% endraw %}

Simple IF...ELIF...ELSE Statement

You can also add an ELIF statement to an IF statement. This will allow you to have multiple else sections. This would allow showing section 1 if the variable is option one, else section 2 if the variable is option 2 or section 3 if the variable is option 3

{% raw %}

{% if data['variable-3'] == 'Option 1' %}

HTML if variable-3 is equal to 'Option-1' ...

{% elif data['variable-3'] == 'Option 2' %} /p>

HTML if variable-3 is equal to 'Option-2' ...

{% else %}

HTML if variable-3 is equal to anything else ...

{% endif %}

{% endraw %}

For loops

A FOR Statement allows you to loop through a list of variables and display or interact with themin turn.

Simple FOR loop

If statements are started with an {% raw %}{% for item in ...%}.{% endraw %} and ended with {% raw %}{% endif %}.{% endraw %}. The word 'item' can be any word, use something that makes sense in the context. You will have to refer to the list item by that word when you refer to it in code.

{% raw %}

{% for item in data['item-list'] %}

... HTML to show for each item in list ...

{% endif %}

{% endraw %}

Example One

Working example (Opens in seperate page).

This example shows how to show a list for each item selected in a checkbox component

In the diagram below we can see that the user has selected items from a checkbox component. We want to show each selected item in a list on another page

The code to do this is as follows

{% raw %}

<h2 class="govuk-heading-m">Selected options</h2>

<hr>

{% for option in data['Options'] %}

<p>{{ option }}</p>

{% endfor %}

{% endraw %}

{% for option in data['Options'] %}

{{ option }}

</div>

<textarea class="govuk-textarea govuk-!-padding-top-5" name="options-text" rows="5"></textarea>

<hr>

{% endfor %}

{% endraw %}

Includes

An INCLUDES statement is a way to include common pieces of code in multiple places without having to re-write it each time

INCLUDES statements are written as -

{% raw %}

{% include "../includes/code-block.html" %}

{% endraw %}

where code-block.html is the common code you want to include. It is standard to keep all INCLUDES blocks in a folder called 'includes'

Filters

Filters allow you to format and change the data variables.

Filters are written as a bar then the filter type. For example if you wanted to Capitalise a variable you would do the following -

{% raw %}

{{ data['item'] | Capitalise }}

{% endraw %}

Many filters are available and are listed here in the Official Jinja documentation.
